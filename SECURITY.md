# Sécurité

Ne publiez pas de vulnérabilité ou de secret dans une issue publique. Retirez
immédiatement toute clé accidentellement exposée et régénérez-la chez le
fournisseur concerné.

Principes du projet : moindre privilège, capture volontaire, API locale,
jetons temporaires, stockage sécurisé des secrets et données locales par
défaut.
